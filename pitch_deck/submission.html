1. Project Description

The problem. Multisig signers approve transactions they cannot read. $2 billion stolen in 18 months across four multisig hacks: Bybit ($1.46B, Feb 2025) — Lazarus pushed malicious JS into Safe's frontend, signers' Ledgers blind-signed a delegatecall the UI hid as a 30,000 ETH transfer. Drift ($285M, Apr 2026) — DPRK social-engineered Squads multisig signers into pre-signing durable-nonce governance transactions they couldn't fully verify on hardware. WazirX ($235M, Jul 2024) — Liminal-managed Safe UI displayed one transaction; signers approved another that swapped the implementation for a malicious contract. Radiant ($50M, Oct 2024) — malware showed a routine config change while hardware wallets signed transferOwnership on the lending pool. Every hack used hardware wallets. Every hack, the wallet couldn't read what was signed. Every hack, trust fell back to a UI that lied.

What Lucid does. A multisig where signers approve human-readable intents via signMessage, not opaque transactions via signTransaction. The Ledger displays the actual action in plain text: approve add market 5 with oracle 9abcD3F2vN5pQ8rR4mT9wF2jB6cL3nE1xY5dG7sH4def | wallet: dao-governance (7Hk2vN5pQ8rR4mT9wF2jB6cL3nE1xY5dG7sH4kP9zU2A); proposal: #42; expires: 09 May 2026 12:00:00 UTC;. Protocols define their governance ruleset as verified intents auto-generated from their program's IDL — each intent specifies the exact CPI, parameters, and human-readable template. Proposers and approvers sign what they read, on the device itself, outside any compromised host.

Why Solana. Lucid exploits Solana's ed25519 precompile for on-chain signature verification of off-chain messages. The program reconstructs the human-readable message from stored intent definitions and proposal parameters, then verifies the signer's ed25519 signature against it — if anything is tampered between proposal and signing, the reconstructed message won't match the signature, and the program rejects the CPI. Sub-cent verification cost. This is only possible because Solana exposes the ed25519 precompile as a native program.

Technical execution. ~6,000 LOC Pinocchio on-chain program (not Anchor) with zero-copy accounts, 11 instructions, and a byte-pool pattern for variable-length intent data. TypeScript SDK with a two-tier verification engine: Tier 1 matches known programs by discriminator, Tier 2 performs structural validation against Anchor IDLs. Full React dashboard with direct Ledger WebHID signing (no wallet-adapter middlemen), wallet creation, proposal tracking, timelock countdowns, and an interactive demo. 217 tests passing across Rust (LiteSVM with precompile support), CLI, SDK (vitest), and dashboard.

What's next. Devnet deployment, integration with one live protocol (targeting governance admin keys), and a signMessage fallback for mobile wallets that can't use Ledger. Intent template hashing on-chain (SHA256 of the intent JSON stored in the account header) is designed but not yet implemented — it would let verifiers confirm intent authenticity without trusting the frontend.

Innovation (25%): Novel primitive — no existing multisig uses signMessage for human-readable approval on hardware wallets. The ed25519 precompile verification of off-chain messages against on-chain state reconstruction is new. Auto-generated intents from IDLs with two-tier verification is new. Closest comparables — Squads and Realms — both use signTransaction, so signers must enable blind-signing on Ledger and see only hex.

Technical Execution (20%): Pinocchio (not Anchor) for zero-copy performance. Byte-pool pattern for variable-length on-chain data. LiteSVM integration tests with real ed25519 signatures verified on-chain. Direct Ledger WebHID — no wallet-adapter middlemen. Clean four-surface separation: program / CLI / SDK / dashboard. ~22K LOC, all tested.

Every one used hardware wallets that couldn't read the transaction. Lucid eliminates the attack vector at the hardware level. Public good: the signing primitive is open and protocol-agnostic. Any program with admin keys can adopt it from the IDL.

Presentation (10%): Direct, technical, no fluff. Architecture diagram in README. Real before/after Ledger recordings (blind-signing.mp4, lucid-signing.mp4) demonstrating the primitive on the device itself. Demo script with tamper detection.
