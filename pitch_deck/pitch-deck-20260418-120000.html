THE SOLUTION

Read what you sign — on the device itself.

Intent Definitions

Anchor IDL → human-readable ruleset. Every action templated.

Plain-English Signing

Signers approve via signMessage. Wallet renders on-device, outside any compromised host.

On-Chain Verify

Program reconstructs the message and verifies the signature. Tamper → reject.

Aligned with sRFC 38: Offchain Message Specification Version 1 — Anza, December 2025.

Speaker Notes
How we solved it? We changed the flow. Before any transaction runs, the wallet defines a ruleset. We call those templates intents — each one carries human-readable text that renders on the hardware wallet. After you sign, the program reconstructs the message and verifies your signature. If anything doesn't match, the chain rejects it.
